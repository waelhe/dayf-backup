# مواصفة: تكامل Discourse + Saleor مع Dayf
**النظام:** البنية التحتية والتكامل
**الفرع:** `004-integration`
**التاريخ:** 2025
**الحالة:** Draft

## قصص المستخدمين (مرتبة بالأولوية)

### P1 — منتدى المجتمع (Discourse)
كـ **مستخدم** أريد **منتدى للنقاشات والمراجعات السياحية** حتى **أتفاعل مع المجتمع وأحصل على نصائح حقيقية**.

**اختبار مستقل:**
- المستخدم يسجل دخوله عبر Dayf SSO
- ينشئ topic جديد
- يرد على موضوع موجود

**سيناريوهات القبول:**
1. Given مستخدم مسجل في Dayf When يزور المنتدى Then يدخل تلقائياً بدون تسجيل جديد
2. Given مستخدم يكتب مراجعة When يضغط نشر Then تظهر في المنتدى والملف الشخصي
3. Given admin When يحذف محتوى Then يُسجل في Audit Log

### P2 — السوق الإلكتروني (Saleor)
كـ **بائع سياحي** أريد **متجر إلكتروني متكامل** حتى **أعرض وأبيع منتجاتي وخدماتي السياحية**.

**اختبار مستقل:**
- البائع ينشئ منتج جديد
- الزائر يضيف للسلة ويكمل الشراء
- الطلب يُسجل في النظام

**سيناريوهات القبول:**
1. Given منتج جديد When يُنشأ Then يظهر في المتجر
2. Given مستخدم يتصفح When يضيف للسلة Then يُحدث السلة
3. Given طلب جديد When يُؤكد Then يُرسل إشعار للبائع

### P3 — Single Sign-On (SSO)
كـ **مستخدم** أريد **تسجيل دخول واحد** حتى **أتنقل بين Dayf والمنتدى والمتجر بدون تسجيل متكرر**.

**اختبار مستقل:**
- تسجيل دخول في Dayf
- زيارة المنتدى → مسجل تلقائياً
- زيارة المتجر → مسجل تلقائياً

**سيناريوهات القبول:**
1. Given مستخدم مسجل في Dayf When يزور Discourse Then يُنشأ session تلقائياً
2. Given مستخدم مسجل في Dayf When يزور Saleor Then يُنشأ user تلقائياً
3. Given تسجيل خروج من Dayf When يُعيد الدخول Then لا توجد sessions معلقة

## المتطلبات الوظيفية

- FR-001: النظام يجب أن يوفر SSO عبر Supabase Auth
- FR-002: المنتدى يجب أن يعمل محلياً على منفذ مستقل
- FR-003: المتجر يجب أن يعمل محلياً على منفذ مستقل
- FR-004: البيانات يجب أن تُخزن في PostgreSQL محلي
- FR-005: التخزين المؤقت يجب أن يستخدم Redis محلي
- FR-006: Webhooks يجب أن تربط الأنظمة مع Dayf

---

## معايير النجاح (قابلة للقياس)

- SC-001: SSO يعمل خلال 3 ثوانٍ كحد أقصى
- SC-002: المنتدى يقبل 10 طلبات/ثانية بدون تأخير
- SC-003: المتجر يقبل 10 طلبات/ثانية بدون تأخير
- SC-004: الذاكرة الإجمالية أقل من 2.5 GB بعد التحسين
- SC-005: جميع APIs تعود بـ 200 OK

---

## الكيانات الرئيسية

- **DiscourseUser**: مستخدم المنتدى (مرتبط بـ Supabase Auth ID)
- **SaleorUser**: مستخدم المتجر (مرتبط بـ Supabase Auth ID)
- **WebhookEvent**: حدث يُرسل بين الأنظمة
- **SSOSession**: جلسة موحدة عبر الأنظمة

---

## الافتراضات

- Dayf يعمل على Supabase (Cloud)
- السيرفر المحلي (Sandbox) هو بيئة التطوير
- لا حاجة لإعداد إنتاجي في هذه المرحلة
- التحسين التدريجي سيتم بعد التثبيت الأساسي
